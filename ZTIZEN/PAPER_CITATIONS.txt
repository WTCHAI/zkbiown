SECTION 1: INTRODUCTION
-----------------------
"Cancelable biometrics [1][2] enable revocable biometric templates through
non-invertible transformations, addressing privacy concerns in biometric
authentication systems. Unlike traditional biometric systems that store raw
templates, cancelable methods [2] allow template revocation and reissuance
when compromised."

"Blockchain technology [11] provides immutable, decentralized storage for
biometric commitments, ensuring tamper-resistance and transparency."

SECTION 2: RELATED WORK
-----------------------
"Privacy-preserving biometric techniques [1] include non-invertible
transformations, biometric cryptosystems, information hiding, and
protocol-based methods. Among these, BioHashing [3][4] transforms biometric
features using user-specific random keys."

"Zero-knowledge proofs [15] enable verification without revealing secrets.
Groth16 [7], a pairing-based non-interactive ZK proof system, offers constant
proof size and verification time, making it suitable for biometric verification."

SECTION 3: METHODOLOGY - BioHashing
------------------------------------
"We implement BioHashing [3] using Gaussian random projection with Gram-Schmidt
orthogonalization, first proposed by Teoh et al. [3]. The algorithm generates
an orthonormal projection matrix Φ ∈ ℝ^(m×n) where each element follows
R[i][j] ~ (1/√m) × N(0,1). Gram-Schmidt orthogonalization ensures statistical
independence between projected bits."

"The composite key K_composite = SHA256(K_platform || K_ztizen || K_user) [17]
seeds a cryptographically secure random number generator (HMAC-DRBG), ensuring
deterministic yet unpredictable matrix generation."

SECTION 3: METHODOLOGY - Poseidon Hash
---------------------------------------
"The binary BioHashing template is hashed using Poseidon [5][6], a
zero-knowledge-friendly hash function designed for algebraic circuits.
Poseidon achieves 0.2-0.5 constraints per bit [5], compared to 1.68 for
Pedersen hash, significantly reducing proof generation overhead."

SECTION 3: METHODOLOGY - ZK Proofs
-----------------------------------
"Template matching is verified using Groth16 zero-knowledge proofs [7], which
allow a prover to demonstrate Hamming distance below a threshold without
revealing the template. The proof system satisfies completeness, soundness,
and zero-knowledge properties [15]."

SECTION 3: METHODOLOGY - Blockchain Storage
--------------------------------------------
"Biometric template commitments are stored on-chain [11], ensuring immutability
and decentralization. Each user receives an NFT-based credential representing
ownership of their biometric template, enabling self-sovereign identity."

SECTION 4: EXPERIMENTAL SETUP
------------------------------
"Face embeddings are extracted using FaceNet [8][13] (128D and 512D) and
ArcFace [9][14] (512D). FaceNet employs triplet loss for metric learning,
while ArcFace uses additive angular margin for improved discrimination.
Face detection and landmark extraction use face-api.js [10]."

SECTION 5: RESULTS - Unlinkability
-----------------------------------
"Our system achieves perfect unlinkability (0.00% cross-key correlation) as
required by cancelable biometric standards [1][2]. Scenario C (same person,
different keys) shows 0% match rate across 5,436 comparisons, demonstrating
effective privacy protection."

WHAT NOT TO CITE (Common Mistakes)
================================================================================

✗ Achlioptas 2003 (Sparse Random Projection)
  Reason: Your code uses Gaussian + Gram-Schmidt, NOT sparse ternary

✗ Dodis et al. 2004 (Fuzzy Extractors)
  Reason: You don't use fuzzy extractors, you use BioHashing

✗ Juels & Sudan 2006 (Fuzzy Vault)
  Reason: You don't use polynomial-based fuzzy vault

✗ Juels & Wattenberg 1999 (Fuzzy Commitment)
  Reason: You don't use error-correcting code commitment

✗ Quantization papers
  Reason: You use binarization (threshold τ=0), not multi-level quantization
